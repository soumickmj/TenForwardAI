Step 3: The Training Loop‚ÄîWhere the Magic Happens ‚ú®

Training is an iterative process where we repeatedly show the model our training data and gradually nudge its weights in the right direction. This entire process is repeated for several Epochs, with each epoch representing one complete pass through the dataset.

üîÑ Inside a Single Epoch (e.g., Epoch 1 of 100)

An epoch is one complete pass through the entire training dataset. To make this computationally manageable and improve convergence, the data is processed in smaller Batches (typically 32, 64, or 128 samples).

üöÄ 3a. Forward Pass

A batch of data flows through the network from input to output. Each layer transforms the data, and the model produces predictions.

predictions = model(input_batch) # Shape: [batch_size, num_classes]
‚Üì

üìä 3b. Calculate Loss

We compare the model's predictions with the actual labels. The 'Loss' quantifies how wrong the model was‚Äîlower is better!

loss = criterion(predictions, true_labels) # Common loss: CrossEntropyLoss for classification
‚Üì

üîÑ 3c. Backward Pass (Backpropagation)

Mathematics to the rescue! We calculate gradients‚Äîthe direction and magnitude needed to change each weight to reduce the loss. This uses the chain rule from calculus.

loss.backward() # Computes gradients for all parameters

‚ö° 3d. Update Weights

The Optimiser (e.g., Adam, SGD, RMSprop) updates the model's weights using the calculated gradients. The magnitude of updates is controlled by the Learning Rate.

optimiser.step() optimiser.zero_grad() # Clear gradients
üéØ The Learning Rate (LR)

A crucial hyperparameter controlling the step size of weight updates. Think of it as walking speed towards the optimal solution.

‚ö†Ô∏è
Too High (e.g., 0.1+):
May overshoot optimal solutions, causing oscillations or divergence
üêå
Too Low (e.g., 0.00001):
Extremely slow convergence, may get stuck in local minima
‚úÖ
Just Right (e.g., 0.001-0.01):
Balanced convergence speed and stability

üîÑ The Loop Continues: Steps 3a‚Üí3b‚Üí3c‚Üí3d repeat for every batch in the dataset. Once all batches are processed, one epoch is complete! The process then repeats for the next epoch.

Step 4: Validation & Progress Monitoring

After each epoch, we evaluate the model on the Validation Set. This provides an unbiased estimate of how well our model generalises to new, unseen data‚Äîthe ultimate test of machine learning success.
